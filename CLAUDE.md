# CLAUDE.md (Router)

이 파일은 **라우터(진입점)** 입니다. 실제 규칙/진행은 아래 파일이 단일 진실(SSOT)입니다.

- 규칙(헌법): `MEMORY.md` (+ 필요 시 MEMORY_* 분할 파일)
- 현재 진행/이슈/다음 할 일: `PROGRESS.md`
- 과거 로그(아카이브): `ARCHIVE_YYYY_MM.md`

작업 시작 순서:
1) CLAUDE.md → 2) MEMORY.md → 3) PROGRESS.md

작업 종료 시:
- PROGRESS.md 업데이트
- `.commit_message.txt` 업데이트
- (환경/제약 변경 시) MEMORY.md의 Constraints 갱신
- PROGRESS가 두꺼워지면 ARCHIVE로 이동(가이드 참고)

가이드(선택):
- 평소에는 MEMORY/PROGRESS만 따른다.
- 운영 이슈(모델 스위칭/충돌/아카이브/캐싱)가 생기면 `AI_HYBRID_GUIDE_v2.2.md`의 **해당 섹션만** 참조한다.

보안:
- 토큰/비밀번호/키스토어/인증서 등 **비밀값은 문서/코드/예시에 절대 쓰지 말 것**

## 완성도 체크 규칙 (필수)

**기능 구현 시, 구조(틀)만 만들고 데이터/내용을 비워두거나 최소한만 채우는 것을 금지한다.**

### 원칙
1. **구조를 만들면 데이터도 100% 채운다** — 템플릿 엔진을 만들었으면, 모든 조합의 메시지를 즉시 채운다. "나중에 추가" 금지.
2. **연동 가능한 코드가 있으면 즉시 연동한다** — 이미 구현된 함수/서비스가 있는데 연결만 안 한 상태를 방치하지 않는다.
3. **작업 완료 전 사용자에게 확인한다** — 다음 질문을 반드시 한다:
   - "이 기능에서 아직 연동하지 않은 기존 코드가 있는지 확인할까요?"
   - "데이터가 빠짐없이 채워졌는지 검증할까요?"
   - "이 작업으로 인해 추가로 해야 할 연쇄 작업이 있는지 확인할까요?"
4. **기존 코드 활용 우선** — 새 기능 구현 시, 프로젝트에 이미 존재하는 관련 서비스/함수를 먼저 탐색하고 연동한다. 있는데 안 쓰는 것은 낭비다.

### 체크리스트 (기능 구현 완료 시 반드시 확인)
- [ ] 모든 데이터 슬롯이 채워져 있는가? (빈 배열, placeholder 텍스트 없는가?)
- [ ] 프로젝트에 이미 있는 관련 함수/서비스 중 연동 안 된 것이 없는가?
- [ ] 연쇄적으로 수정해야 할 파일이 남아있지 않은가?
- [ ] 사용자가 앱에서 실제로 볼 수 있는 상태인가? (화면에 표시되는가?)
